## Cómo ejecutar la configuración automática

1. Ve a **Actions** → **Configurar Tablero de Proyecto por Sprints**
2. Haz clic en **Run workflow**
3. Ingresa el **número de tu GitHub Project** (visible en la URL: `github.com/users/StivenGs95/projects/N`)
4. Haz clic en **Run workflow** para ejecutar

El workflow automáticamente:
- ✅ Creará todas las etiquetas por sprint y tipo
- ✅ Creará los 18 issues con títulos, descripciones y criterios de aceptación
- ✅ Agregará todos los issues a tu GitHub Project

## Metodología ágil aplicada

- **Framework:** Scrum adaptado a proyecto académico
- **Duración de sprint:** 2 semanas
- **Ceremonia de inicio:** Sprint Planning con backlog refinado
- **Ceremonia de cierre:** Sprint Review + Retrospectiva
- **Herramienta:** GitHub Projects (tablero Kanban con iteraciones)
